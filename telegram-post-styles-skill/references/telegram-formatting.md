# Форматирование Telegram

Правила, которые агент обязан соблюдать при написании постов.

## Синтаксис

| Элемент | Синтаксис | Когда использовать |
|---|---|---|
| Жирный | `*текст*` | Заголовки блоков, ключевые цифры, важные фразы |
| Курсив | `_текст_` | Акценты, цитаты, внутренние мысли |
| Моноширинный | `` `текст` `` | @username, ключевые слова для CTA, команды |
| Ссылка | `[текст](https://url)` или голый URL | Каналы, внешние ресурсы |
| Спойлер | `\|\|текст\|\|` | Интрига, скрытый вывод (редко, 0–1 на пост) |
| Зачёркнутый | `~текст~` | Устаревшие подходы, «было» в кейсах |

## Структура и читаемость

- Пустая строка между смысловыми блоками.
- Абзацы по 1–3 строки — не писать «простыни».
- Хук (первая строка) — короткий, цепляющий, виден в превью канала.
- Списки: нумерованные `1.` или маркеры `—` / `➡️` / `⏺` в зависимости от стиля.
- Диалоги: тире `—` в начале реплики, имя собеседника через двоеточие.

## Эмодзи

- 3–8 эмодзи на пост как визуальные якоря.
- В начале поста — 0–1 эмодзи в заголовке.
- В списках — по одному на пункт (➡️, ⏺, 1️⃣, ✅).
- Не ставить эмодзи подряд больше двух.
- Не заменять ими смысл — только усиливают акцент.

## Хештеги и опросы

- Хештеги: только в официальном и дайджест-стиле, 5–7 штук в конце поста (перед футером).
- Опросы (текстовые варианты): в официальном стиле, формат `— ✅ Вариант`.
- Формат: `#Авито #Нейросети #Маркетинг` — с заглавной буквы, без пробелов.

## Чего избегать

- Markdown-заголовки `#` — в Telegram не рендерятся.
- HTML-теги — не использовать.
- Двойные звёздочки `**` — в TG жирный через одну `*`.
- Слишком длинные посты без разбивки (более 15 экранов на телефоне).
- КАПС LOCK целиком — только для 1–2 слов в хуке.

## Проверка перед выдачей

- [ ] Хук читается за 2 секунды.
- [ ] Есть визуальные разрывы (пустые строки).
- [ ] Форматирование совместимо с нативным редактором TG.
- [ ] @username и ключевые слова в `моноширинном` где уместно.
- [ ] Футер каналов добавлен после пустой строки.
